This is a PR review task. Do NOT write a plan, implement code changes, or call /wrap-up — you are here to review the PR, not to implement anything.

1. Find the PR number or URL from the task description.
2. Check the diff size: `gh pr diff <PR-number> --repo <owner/repo> | wc -l`
3. Choose the review command based on diff size:
   - Small PR (< ~300 lines changed): run `/review`
   - Large PR (≥ ~300 lines changed): run `/review-pr`

After running the review, present your findings and wait for the user's instructions. Do NOT call /wrap-up.
